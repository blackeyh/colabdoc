The repo also closes the biggest earlier architecture gap by moving live collaboration from whole-document overwrite behavior to a Yjs-backed CRDT flow while keeping a simple FastAPI deployment story.

Architecture

Final System Design

The shipped architecture is intentionally simpler than a production multi-service platform, but it is now fully documented and internally consistent.

Architecture Snapshot

Design Decisions

Backend-managed authentication

Local JWT auth replaced the earlier external-provider plan to improve reviewer setup and testing.

Yjs in the browser

Realtime collaboration now merges connected edits through a CRDT path while keeping the server deployment lightweight.

Canonical JSON persistence

The backend still stores document JSON for export, AI context, and version history.

Provider abstraction for AI

The same backend routes support a local deterministic fallback or a real OpenAI-compatible provider.

Implemented Feature Set

Authentication & Access

Registration, login, JWT bearer auth, refresh flow, and protected document access are implemented server-side.

Document Management

Create, rename, load, edit, delete, and persist rich-text documents with a compatibility layer for legacy content.

Sharing & Roles

`owner`, `editor`, `commenter`, and `viewer` roles are enforced in both the UI and backend endpoints.

Realtime Collaboration

AI Assistant

Streamed AI responses, cancellation, compare-before-apply, accept/edit/reject, history, and undo after acceptance are supported.

Versions & Restore

Version snapshots can be created, listed, and restored, with reset propagation to connected collaborators.

Export

Reviewer Setup

Documented Trade-Offs

Auth Storage

Tokens remain in localStorage for simplicity and WebSocket compatibility.

Realtime Persistence

The latest Yjs room snapshot is kept in memory rather than a dedicated multi-instance backend.

AI Provider

The default local config uses a deterministic provider unless a real OpenAI-compatible model is configured.
